## 0. 这个任务干什么、不干什么
干：对 13 个无合格墓碑的欠账格，扫全球（含中文世界）死亡/衰败公司，按下方验收清单逐个分档，把结果写进暂存表和近邻物料独立文件。

- 不碰任何 CELL-NNN 文件的 ## 墓碑 / 生死比表。生死比表是受保护表，本任务全程只读不写。严格同格墓碑入表发生在 Claude 终判之后，不在本任务。
- 不改任何格子的状态标记、不改 _分类总表.md、不改 空格子-预测库.md。状态升级（欠账→脆弱/稳健）依赖终判，是终判后的事。
- 不拍终判。Codex 只填初判，终判-合格 / 终判-死法档 两列留空给 Claude。
- 不讨论"中国能不能照着做产品赚钱"（SK 的活，越界即停）。

## 2. 验收清单（对每个候选死者逐条机械执行）
每发现一个候选死亡/衰败公司，按四道闸顺序判定，结果写进暂存表一行。

闸1·准入（只卡两根稳定轴：层 + 踏脚石）
- 同层？候选押注的位置属于本格 L 层。
- 踏脚石严格同义？候选押注的使能前提与本格踏脚石原文是同一件事的同义表述，不是"赛道接近"。
  正例：本格踏脚石"联网检索、引用归因与生成式回答组合成可用搜索体验"，候选做生成式答案搜索 → 严格同义。
  反例：本格"碳市场项目质量评级与监测核验"，候选做"海洋工程碳移除" → 赛道接近但踏脚石不同 → 不准入（判孤儿，参考 TASK-004 Running Tide）。
- 象限不设门槛。不要用"是不是同象限"筛候选——象限是待检验假设，墓碑死法是反推校正象限的工具，拿象限当门槛是循环论证。象限留到死法机制出来后由 Claude 校正。
- 判定写入 初判-严格同格 列：Y / N。N 且死亡属实 → 转闸4当近邻物料处理；并判断它是否是别的某个 CELL 的严格同格墓碑（是则标建议归格），否则归孤儿。

闸2·证据（死亡/衰败事实的可核查性）
- 死亡/衰败判定（满足其一）：已关停 / 被贱卖（成交价<累计融资）/ ARR 腰斩或长期停滞 / 估值腰斩 / 创始团队官宣放弃原方向。
- 证据级：A 一手财报或官宣 / B 权威媒体 / C 第三方估算或传闻 / X 仅传言。
- 只有 C 或 X 的：记录但标"证据不足"，不进严格墓碑候选（C/X 不能清欠账）。
- 每个候选必带：公司名、国家、成立年、累计融资、巅峰估值或 ARR、死亡/衰败时间、来源链接、证据级。

闸3·死法分档（结构性 vs 特异）—— Codex 只提初判
- 写一句话死法机制（不是"失败了"，是"这个结构位置为什么撑不住"）。
- 初分两档，填 初判-死法档 列：
  结构性：死法威胁踏脚石本身（被底层模型一次升级吞掉 / 单点功能无护城河 / demo 惊艳但留存崩 / CAC 长期>LTV / 踏脚石根本没真正跨阈值）。
  特异：死法是个案、不威胁踏脚石（财务造假 / 治理崩盘 / 创始人内讧 / 单一监管个案 / 纯资本环境）。
- 初判是建议，不是终判。终判-合格 和 终判-死法档 两列留空，由 Claude 填。

闸4·近邻物料归档（过闸2死亡属实、但闸1=N 的）
- 写进独立文件 格子档案/近邻物料-非生死比.md（见 §3），不写进任何生死比表、不写进暂存表严格候选区。
- 这些只供将来机制提炼复用，永不进生死比。
