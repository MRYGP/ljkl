## 纪律（每一步都适用）
- **踏脚石必须可定日、可引用。** 「什么刚翻转」要给出具体翻转点（能力/成本/政策/行为）+ 日期 + 来源链接。给不出 → 该候选保持 `观察/欠账`，不入预测。不准用「邻近可见」「大致成熟」搪塞。
- **无墓碑不入档（铁律二）。** 任何要进预测库的格，必须带生死比，且标口径（同格 / 近邻对照），沿用 CELL-001 的诚实写法。补不到 → 标 `status: open` 欠账，列入待补清单。
- **未找到是合法结果。** 不造案例、不硬凑数量、不为完成 TASK 升级证据等级。
- **右下（暗牌×羞耻/合规锁）只抽机制不下场。** `redline: true`、标题前加 🔴、正文只记可迁移机制，删除一切平台名 / 路径 / 可复制操作细节。
- **不功利（铁律三）。** 任何文件不得出现变现、增长、回报期限。
- 证据分级 A/B/C/X，X 仅作灵感不作判断依据。
- 编辑纪律：只改本任务直接触发的部分，不重写无关段落。

### B1 太早死死者 → 还空着的相邻格（锚定 ORPHAN-001 Google Glass）
Glass 已标「太早死-奇点指针 / lead_gap 111 个月 / 混合社会锁」，踏脚石（消费级眼镜接入多模态）已于 2024-04 翻转，Ray-Ban Meta 点亮了**相邻**格。关键问题不是 Glass 为什么死，是：

执行：
1. 核查 Ray-Ban Meta（及同类）实际点亮了哪个具体需求格（可核查的能力边界，给来源）。
2. 用差集找出 Glass 当年瞄准、但现有产品**仍未覆盖**的相邻空格，按坐标系定位（层 × 地域 × 象限）。

4. 扫该空格的同格或近邻墓碑，算生死比。
5. 够条件 → 写成 `空格子-预测库.md` 一条预测（按文件顶部写法规约：可证伪 + 时间窗 + 可观测验证信号 + 下次对账日）。不够 → 列欠账，写明差哪一块（多半是踏脚石翻转点或墓碑）。
