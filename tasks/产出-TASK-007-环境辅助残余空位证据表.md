# TASK-007｜主动环境辅助残余空位证据表
> 版本：v1.0 · 2026-06-03
> 口径：只核查 Glass / P-003 严格差集，不扩展到其它环境智能格子；未找到即记未找到，不立 P-004。

## 结论
不立 P-004。严格靶心「无需显式唤起、可全天佩戴、隐私边界可被社会接受的主动环境辅助」截至本轮仍未被公开产品完整满足，但三条腿没有同时成立。

本轮最重要发现：这格主导门槛不是单纯技术锁，而是社会接受度 / 隐私锁。技术锁和成本锁常有可定日翻转点；社会接受度更像渐进正常化，未必存在某一天的「刚翻转」。因此，本项目当前用「可定日踏脚石」驱动预测的仪器，对社会锁格子可能失灵。待人决定：社会锁格子是否需要改用正常化领先指标，还是一律保持观察位、不立预测。

## 信号体检
目标信号必须三者同时成立：
- 无需显式唤起：不是每次说 `Hey Meta`、按按钮、抬手或捏合后才工作。
- 可全天佩戴：不是短会话、演示或会议记录器，而是日常背景存在。
- 隐私边界可被社会接受：旁人知道或可合理感知边界，不把常驻摄录当作冒犯。

核查结论：未发现三者同时成立的公开产品。

已接近但未填满：
- Ray-Ban Meta / Meta Ray-Ban Display：覆盖按需视觉 AI、跟进问答、短时 Live AI、翻译、导航和 EMG / 神经带交互，但仍需初始唤起或显式控制，且 Live AI 不是全天背景主动辅助。[Meta 2024-09](https://about.fb.com/news/2024/09/ray-ban-meta-glasses-new-ai-features-and-partner-integrations/)、[Meta 2025-09](https://about.fb.com/news/2025/09/meta-ray-ban-display-ai-glasses-emg-wristband/)
- Ray-Ban Meta 的「Respond without Hey Meta」只是在一次请求后的短时间继续听，官方 FAQ 明确会在几秒后自动关闭；它证明需求与产品方向存在，不证明无唤起全天主动已成立。[Ray-Ban FAQ](https://www.ray-ban.com/usa/c/frequently-asked-questions-ray-ban-meta-smart-glasses)
- Bee、Limitless、Friend、Plaud 等更偏音频记录、个人记忆或陪伴，不满足「视觉/环境 + 社会可接受 + 主动辅助」三者。

## 三条腿

### 1. 第三方需求痕迹：有，但证据等级仅 C/B-
- Ray-Ban FAQ 已提供「Respond without Hey Meta」设置，说明减少重复唤起是被产品承认的需求；但它只覆盖短 follow-up，不覆盖全天主动。
- Engadget 对 2025-11 Meta 眼镜快捷功能的报道指出，反复说 `Hey Meta` 在公共空间有尴尬感；这是一条第三方需求痕迹，指向「不想显式唤起」。[Engadget](https://www.engadget.com/wearables/metas-smart-glasses-have-a-new-shortcut-to-call-and-text-without-saying-hey-meta-213303178.html)
- Reddit 用户长期问「能否不每句都说 Hey Meta」「如何不说 Hey Meta 激活」，可作为 C 级影子用法，不单独升级判断。

判定：需求痕迹成立，但还不够支撑预测。它证明「少唤起」需求，不证明旁人接受全天主动摄录。

### 2. 可定日踏脚石：缺口成立，且可能是结构性缺口
候选踏脚石均只能补一部分：
- 2024-09，Ray-Ban Meta 公布 follow-up 免重复 `Hey Meta`、记忆停车位置、实时翻译等功能：补「低摩擦交互」，不补全天主动。[Meta 2024-09](https://about.fb.com/news/2024/09/ray-ban-meta-glasses-new-ai-features-and-partner-integrations/)
- 2025-09，Meta Ray-Ban Display + Neural Band 公布：补「低调输入 / 显示」，不补旁人对常驻摄录的接受。[Meta 2025-09](https://about.fb.com/news/2025/09/meta-ray-ban-display-ai-glasses-emg-wristband/)
- 2025-07，Amazon 收购 Bee：证明音频常驻助手仍被押注，但报道标题与讨论本身都把「记录你说的一切」推到隐私争议中心，反而是社会锁反例。[TechCrunch](https://techcrunch.com/2025/07/22/amazon-acquires-bee-the-ai-wearable-that-records-everything-you-say/)

判定：未找到「隐私可接受」的可定日翻转点。更准确地说，社会接受度 / 隐私锁可能不是单点翻转，而是渐进正常化；这需要另一套预测仪器，不能硬塞进当前踏脚石格式。

### 3. 墓碑 / 生死比：有近邻对照，缺严格同格
严格同格口径：活 0 / 死 0。没有公开产品完整押注「无唤起 + 全天主动 + 隐私可接受的环境辅助」并完成市场验证或死亡。

## 象限判定
坐标：L4 体验重塑 × 全球观察 × 左下（需求明牌 / 门是社会接受度与隐私锁，兼技术锁）。

依据：
- 需求是明牌：随身助手、减少唤起、双手占用时获得帮助都是公开需求。
- 门的主导锁由死法决定：Google Glass 的消费场景死法指向隐私反弹与社会污名；Bee / Friend 的反应继续指向旁人接受度，而不是单纯模型能力。
- 本格不是右下红线：需求不是暗牌羞耻需求，因此保留公司名、死者名和来源。

## 何时重开
出现以下任一类信号，再重开 P-003 残余空位；否则保持观察位，不立 P-004：
- 某主流眼镜或轻量可穿戴在公开说明中默认提供「无唤起、持续背景环境理解、主动提醒」，且不依赖短会话 Live AI。
- 产品明确给出旁人可理解的隐私边界，例如本地处理、可见录制状态、空间/对象级采集限制、第三方同意机制，并获得真实日常使用而非演示场景。
- 第三方场景出现持续使用证据：学校、医院、办公室、公共交通等高敏感环境没有禁用，或已有清晰接受规范。
- 能建立「正常化领先指标」口径后，再用该仪器判断社会锁是否跨阈值。

## 待决问题
社会锁格子是不是该用另一套预测仪器：正常化领先指标，而非可定日踏脚石？如果不改仪器，本格应长期挂「观察位、不预测」，避免把渐进社会过程伪装成单日翻转。
